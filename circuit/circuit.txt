
Power Circuit with Soft Power Switch

lipo-2s    : 7.4 volts nominal lipo 2s 400mAh battery
             min max voltage range     6.0 - 8.4 volts
             recommended voltage range 6.6 - 8.4 volts

r1         : 20k resistor, top part of voltage divider
Vsense     : connects to analog gpio pin of microcontroller
             to measure battery voltage
             VsenseVolts = 3.3 * digitalRead( VsensePin ) / 4048
             battery voltage = VsenseVolts * ( r1 + r2 ) / r2
             battery percent can be estimated using a voltage to percent table
r2         : 10k resistor, bot part of voltage divider
r5         : 1k resistor for led to indicate state of gpio pin used to
             keep the dc-dc converter enabled.  It will change as soon as
             the time of the long press required to turn on or off the
             microcontroller has been reached, at which point the soft
             power switch can be released.  Note: other values can be used
             for this resistor such as 220 ohm for brighter or 2k for dimmer.
led        : the enable hold indicator led
t1         : 2N7000 n-channel mosfet, used to turn on led and enable the
             voltage divider circuit used to check battery voltage.  This allows
             the use of smaller transistors in the voltage divider without the
             penalty of drawing power from the battery when the system is not in use.
             I used this transistor as it was the most suitable one I have.
r6         : 10k resistor, ensures that the gate is off when the led/vsense enable signal is off

r3         : 100k resistor, bot part of voltage divider that
             includes internal components within dc-dc buck
             converter, required to ensure voltage of enable
             pin does not exceed the microcontroller max gpio
             voltage
             value is high enough that the default state of
             the enable pin is enabled

enable     : when high, the dc-dc buck converter is enabled and
             everything powered by Vout will therefore be on.
             It appears this pin requires a minimum of 1 volt to be
             enabled.
             WARNING:  It is important to note that this pin will output
             a voltage a little under the voltage of the power source
             going into the Vin.  This voltage is higher than the typical
             limits of GPIO pins, which can damage the microcontroller.
             In order to bring this pin within safe limits, r4, a 100k resistor
             will lower this voltage as it combines with internal circuitry
             to form a voltage divider.  During testing, I believe the internal
             circuits consist of a diode plus a ~ 220k resistor.
r4         : 10k resistor, in testing, the buck converter
             enable pin appeared to go low with any
             value of resistor <= 40k connected to ground
             The 10k value was chosen as it is well within
             the required value to disable the converter.
             When combined with the normally open switch,
             this changes the default state of converter to
             disabled, ensuring the buck converter will draw
             the minimum current from the battery.  This current
             combined with the Vsense voltage divider draws
             about .05 mA in this state, so a fully charged
             battery will take almost a year to discharge.
n/c switch : normally closed momentary switch, press to disconnect r4
             which enables the buck converter and powers on
             the microcontroller.  It is then up to the microcontroller
             to hold the enable pin high so the power stays on after the
             this switch is released.  The switch is also monitored by
             the microcontroller, so that the microcontroller can also
             shutdown the system by releasing the hold on the enable pin.

Vout       : this pin supplies 5 volts to the microcontroller.  In the case
             of the Pi Pico, the Vsys pin can be connected to Vout using a
             Schottky diode such as the 1N5817.  This allows the usb to be
             connected at the same time as Vout is on, without voltage from
             either source feeding into the other power source.  This added
             diode stops usb power flowing to both the dc-dc converter and
             the servos.  The pi pico has an internal diode to prevent voltage
             going into Vsys from feeding to the usb port.
1N5817     : Schottky diode, note the polarity

c1         : 470 mF electrolytic capacitor connected at end of servo power bus
c2         : 470 mF electrolytic capacitor connected near microcontroller Vsys

NOTE 1: I haven't actually tested the voltage divider circuit with the high value resistors in this test circuit
as my test pico has non-functioning analog inputs.  So the only tested divider circuit for battery voltage sensing
is the one currently in use in my currently functioning robot, which uses 20k and 10k resistors.  There definitely
could be issues with using such high valued resistors, so I will have to definitely test this in the near future.

NOTE 2: Circuit updated to include a mosfet transistor to enable the led and vsense circuitry.  Reduces drain on battery
when system is disabled.

Raspberry Pi Pico 2W

Note: The Raspberry Pi Pico W may not work in this configuration as it can only
support 7 servos using the servo.h library I am using.  There are apparently ways
to use many more servos if not using this particular library.
